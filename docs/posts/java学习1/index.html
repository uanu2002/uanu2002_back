基础知识

Java特点

  • 面向对象：将需求分类实现。（面向过程：将需求分分布实现）
  • 安全性高：不支持指针类型、自动垃圾回收、字节码验证、异常处理机制。
  • 结构特点：Java项目的结构为：项目 -> 包 -> 源代码。每个项目至少包含一个类，类中包含方法。
  • …

Java执行过程

.java文件 -> .class字节码文件 -> 运行

1// 代码运行main.java
2// step 1
3javac main.java // javac.exe进行编译并生成main.class字节码文件
4// step 2
5java main // java.exe翻译执行main.class

Java安装

1、下载安装JDK

JDK是Java开发工具包，即提供上述需要的javac.exe、java.exe等运行环境和各种类库和工具。

JDK官网下载

2、配置环境变量

右键此电脑选择属性 -> 高级系统设置 -> 高级 -> 环境变量 -> 在系统变量（用户变量也行）进行以下操作：

  • 设置JAVA_HOME的值为JDK的安装目录。
  • PATH添加java.exe和javac.exe的位置。
  • 设置CLASSPATH的值为类库的路径。

此时就可以用记事本直接编写Java程序，然后更改后缀为,java，再使用上述执行方法即可。

（可选）3、下载安装IntelliJ idea

Intellij idea类似vscode，提供集成开发环境，可以更方便的开发和管理。

Intellij官网下载

Java概念

方法：类比C语言中的函数。

实例变量：类比C语言中的变量。

类：提供一个模板，里面包含实例变量、方法等。eg：人类：有头发，会说话。

对象：类具化的结果。eg：人类的小明：有黑头发，会说北京话。

命名：

  • 大小写敏感。
  • 类名首字母应该大写。
  • 方法名小写开头。
  • 源文件名必须与类名相同。
  • 程序都必须由public static void main(String[] args)的方法开始执行。

  • 初始化：new创建对象时调用构造方法来初始化对象。

11}

源文件声明规则

  • 一个源文件中只能有一个public类
  • 源文件名称要与public类的类名保持一致
  • 类如果定义在包里，package语句应该在源文件的首行
  • import语句要放在package语句和类定义之间，如果没有package语句，import语句要放在源文件最前面。

用于方法中的局部变量和类中成员变量同名，但在方法中需要覆盖（修改）这个成员变量时，需要使用this作为前缀指向类中的成员变量。

示例：

1public class human{
2	String name;

5	}
6}

变量

分类：

  • 局部变量：在类的方法、构造方法或者语句块中定义的变量，方法结束后会自动销毁。
  • 类变量：即静态变量。独立于方法外的变量，用static修饰。
  • 成员变量：即实例变量。独立于方法外的变量，没有static修饰。

局部变量

  • 不能使用访问修饰符
  • 使用栈的内存
  • 没有默认值，必须初始化
1public class Test{
2	public void name(){
3        int age = 20; // 局部变量，只能在name函数内访问。必须要初始化！
4    }
5}

实例变量

  • 对象实例化后，实例变量的值就会确定。
  • 生存周期和对象相同。
  • 可以声明在使用前或后。
  • 可以使用访问修饰符。
  • 有默认值。
1public class test{
2    public  String name; // 实例变量，有默认值。
3}

类变量

  • 初始化后不可改变，常声明为常量。
  • 建议大写。
1public class test{
2    static final String type = "human"; // 类变量。
3}

常用方法

序号 方法和说明
1 public static int binarySearch(Object[] a, Object key) 用二分查找算法在给定数组中搜索给定值的对象(Byte,Int,double等)。数组在调用前必须排序好的。如果查找值包含在数组中，则返回搜索键的索引；否则返回 (-(插入点) - 1)。
2 public static boolean equals(long[] a, long[] a2) 如果两个指定的 long 型数组彼此相等，则返回 true。如果两个数组包含相同数量的元素，并且两个数组中的所有相应元素对都是相等的，则认为这两个数组是相等的。换句话说，如果两个数组以相同顺序包含相同的元素，则两个数组是相等的。同样的方法适用于所有的其他基本数据类型（Byte，short，Int等）。
3 public static void fill(int[] a, int val) 将指定的 int 值分配给指定 int 型数组指定范围中的每个元素。同样的方法适用于所有的其他基本数据类型（Byte，short，Int等）。
4 public static void sort(Object[] a) 对指定对象数组根据其元素的自然顺序进行升序排列。同样的方法适用于所有的其他基本数据类型（Byte，short，Int等）

修饰符

分类

  • 访问修饰符
  • 非访问修饰符

访问修饰符

访问修饰符 使用对象 访问权限
default(即什么都不写) 类、接口、变量、方法 包内、子孙类（包内）
public 类、接口、变量、方法 均可
private 变量、方法 当前类
protected 变量、方法 包内、子孙类（包内）

基类：即父类。 子类：即子类。

非访问修饰符

非访问修饰符 使用对象 作用
static 类方法、类变量 实例化对象时，只有一份拷贝。
final 类、方法、变量 类不能被继承，方法不能被重新定义，变量不能修改。
abstract 类、方法 创建抽象类和抽象方法。
synchronized/volatile 线程的编程。

抽象类：不能用于实例化对象，用于将来对该类进行扩充（类不能同时被abstract和final修饰）。包含抽象方法的类一定要声明为抽象类（抽象类可以不包含抽象方法）。 抽象方法：一种没有任何实现的方法，实现由子类提供。不能被final和static修饰。

static修饰函数：

  • 在类中：即静态成员函数，调用这个函数不会访问或修改任何对象数据成员。
  • 在类外：即静态函数，只能在本文件中被调用，其他方法默认为extern。

static与final区别：

  • static更侧重于作用域，只会被创建一次，可以共享数据。即多个同类只有一个共用的static。
  • final类比于C/C++中的const，定义常量，不能被修改。
  • public完全是作用域，为所有类和方法都能访问。
