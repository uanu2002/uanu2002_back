「Trouble Shooting」某版本GLIBCXX缺失

Posted by Uanu on Mon, Jan 1, 0001

报错

1ImportError: /usr/lib/x86_64-linux-gnu/libstdc++.so.x: version `GLIBCXX_x.x.xx‘ not found(required by xxx)

报错原因：默认路径下的libstdc++.so.x缺少GLIBCXX_x.x.xx

解决方法

  1. 查看目前已有GLIBCXX版本（可选，主要确认是否真的缺失）

    1strings /usr/lib/x86_64-linux-gnu/libstdc++.so.x | grep GLIBCXX

  2. 查找同类型文件

    1sudo find / -name "libstdc++.so.x*"

    选择其中版本较高的，然后复制完整路径名。

  3. 查看是否包含所需GLIBCXX

    1strings 复制的完整路径名 | grep GLIBCXX

    如果包含所需版本则继续下一步，否则在步骤二中重新选择。

  4. 复制到默认目录

    1cp 复制的完整路径名 /usr/lib/x86_64-linux-gnu/

  5. 删除原链接

    1sudo rm /usr/lib/x86_64-linux-gnu/libstdc++.so.x

  6. 建立新链接

    1sudo ln -s 复制的完整路径名 /usr/lib/x86_64-linux-gnu/libstdc++.so.x

然后重新运行问题程序即可。
