Linux(1)--shell语法

Posted by Uanu on Mon, Jan 1, 0001

简介

shell是我们通过命令行与操作系统沟通的语言。shell脚本可以直接在命令行中执行，同时方便复用。 因此，终端中的命令行可以看成是一个“shell脚本在逐行执行”。Linux中常见的shell脚本有很多种，常见的有：

Bourne Shell(/usr/bin/sh``或``/bin/sh) Bourne Again Shell(/bin/bash) C Shell(/usr/bin/csh) K Shell(/usr/bin/ksh) …

在编写脚本时需要制定脚本解释器，例如bash脚本需要再文件开头写#! /bin/bash。

运行方式

作为可执行文件

1:~$ chmod +x test.sh  #chmod +x使得脚本具有可执行权限
2:~$ ./test.sh #在当前路径下执行脚本
3:~$ /home/uanu/test.sh #在绝对路径下执行脚本

使用解释器执行（不需要执行权限）

1:~$ bash test.sh #在当前目录下用解释器执行脚本

默认变量

执行shell脚本时，可以向脚本传递参数，$1是第一个参数，$2是第二个参数，以此类推。$0是文件名（包含路径）。

其他变量

参数 说明
$# 代表文件传入的参数个数
$* 由所有参数构成的用空格隔开的字符串，如"$1 $2"
$@ 每个参数分别用双引号括起来的字符串，如"$1" "$2"
$$ 脚本当前运行的进程ID
$? 上一条命令的退出状态（注意不是stdout，而是exit code）。0表示正常退出，其他值表示错误
$(command) 返回command的stdout（可嵌套）
`command` 返回command的stdout（不可嵌套）

双引号中语句会执行，会获得变量的值。

2echo '$var blog' #输出：$var blog
3echo "$var blog" #输出：Uanu blog

提取子串

数组

区别于C语言的数组，shell语法中的数组可以存放多个不同类型的值，且支持一维数组，初始化不需要指明数组大小。下标从0开始。

数组变量定义

数组用小括号表示，元素之前用空格隔开。

1array=(1 abc "Uanu" xxx)

可以定义单独定义数组中某个元素的值

1array[0]=1
2array[1]=abc
3array[10000]="Uanu"

数组实际的大小并不由下表决定，而是由定义的元素个数决定，下标只作为索引使用。如上定义的数组大小其实为3。

数组读取

元素读取

1${array[index]}

整个数组读取

1${array[@]} # 第一种方式
2${array[*]} # 第二种方式

长度读取

1${#array[@]} # 第一种方式
2${#array[*]} # 第二种方式

expr命令

expr命令用于表达式求值

1expr 表达式

表达式说明

  1. 要用空格隔开每一项
  2. 需要用反斜杠\放在特定字符前
  3. 对包含空格和其他特殊字符的字符串要用引号括起来
  4. expr会在stdout中输出结果。若为逻辑关系表达式，若结果为真，stdout为1，否则为0
  5. expr的exit code：若为计算则为真。若为逻辑关系表达式则取决于逻辑关系表达式真假。注意：exit code真时为0，假时为1。

stdout可以看做C语言中的输出，exit code可以看做C语言中的返回值return。

表达式分类

字符串表达式

  • length str

    返回str的长度

  • index str charset

    charset中任意单个字符在str中最前面的字符位置，下标从1开始。如果str中不存在charset中的字符，则返回0。

  • substr str position length

    返回str从position开始，长度最大为length的子串。如果position或length为负数、0或非数值，则返回空字符串。

示例：

1str="Hello World!"
2
3echo `expr length "$str"`  # ``不是单引号，表示执行该命令，输出12
4echo `expr index "$str" aWd`  # 输出7，下标从1开始
5echo `expr substr "$str" 2 3`  # 输出 ell

整数表达式

算法表达式优先级低于字符串表达式，高于逻辑关系表达式。

  • + -

    加减运算，两端参数会转换为整数，如果失败则报错。

  • * / %

    乘、除、取模运算。两端参数会转换为整数，如果转换失败则报错。

  • ()

    可以表示优先级，但需要反斜杠转义。

示例：

1a=3
2b=4
3
4echo `expr $a + $b`  # 输出7
5echo `expr $a - $b`  # 输出-1
6echo `expr $a \* $b`  # 输出12，*需要转义
7echo `expr $a / $b`  # 输出0，整除
8echo `expr $a % $b` # 输出3
9echo `expr \( $a + 1 \) \* \( $b + 1 \)`  # 输出20，值为(a + 1) * (b + 1)

逻辑关系表达式

  • |

    如果第一个参数非空且非0，则返回第一个参数的值，否则返回第二个参数的值，但要求第二个参数的值也是非空或非0，否则返回0。如果第一个参数是非空或非0时，不会计算第二个参数。

  • &

    如果两个参数都非空且非0，则返回第一个参数，否则返回0。如果第一个参为0或为空，则不会计算第二个参数。

  • < <= = == != >= >

    比较两端的参数，如果为true，则返回1，否则返回0。”==”是”=”的同义词。”expr”首先尝试将两端参数转换为整数，并做算术比较，如果转换失败，则按字符集排序规则做字符比较。

  • ()

    可以表示优先级，但需要反斜杠转义。

示例：

 1a=3
 2b=4
 3
 4echo `expr $a \> $b`  # 输出0，>需要转义
 5echo `expr $a '<' $b`  # 输出1，也可以将特殊字符用引号引起来
 6echo `expr $a '>=' $b`  # 输出0
 7echo `expr $a \<\= $b`  # 输出1
 8
 9c=0
10d=5
11
12echo `expr $c \& $d`  # 输出0
13echo `expr $a \& $b`  # 输出3
14echo `expr $c \| $d`  # 输出5
15echo `expr $a \| $b`  # 输出3

read命令

read命令用于从标准输入中读取单行数据，当读到文件结束符ctrl+d时，exit code为1，否则为0。

参数说明：

  • -p：后面可以接提示信息。
  • -t：后面跟秒数，定义输入字符的等待时间，超过等待时间后会自动忽略此命令，执行后续命令。

示例：

1read name  # 读入name的值
2Uanu  # 标准输入
3echo $name  # 输出name的值
4Uanu  #标准输出
5read -p "Please input your name: " -t 30 name  # 读入name的值，等待时间30秒
6Please input your name: Uanu  # 标准输入
7echo $name  # 输出name的值
8Uanu  # 标准输出。

判断

逻辑运算符&&和||

  • &&表示与，||表示或

  • 二者都具有短路原则

    expr1 && expr2：当expr1为假时，直接忽略expr2

    expr1 || expr2：当expr1为真时，直接忽略expr2

逻辑运算符很多时候可以起到判断语句的作用。

test命令

ps：可用查询命令用法

1man command

test命令用于判断文件类型和变量比较

test命令用exit code返回结果。

示例：

1test 2 -lt 3  # 为真，返回值为0
2echo $?  # 输出上个命令的返回值，输出0
3test -e test.sh && echo "exist" || echo "Not exist"
4exist  # test.sh 文件存在
5test -e test2.sh && echo "exist" || echo "Not exist"
6Not exist  # testh2.sh 文件不存在
类型判断参数 代表意义
-e 文件是否存在
-f 是否为文件
-d 是否为目录
权限判断参数 代表意义
-r 文件是否可读
-w 文件是否可写
-x 文件是否可执行
-s 是否为空文件

格式：

1test -参数 filename
整数比较参数 代表意义
-eq （前者）是否等于（后者）
-ne 是否不等于
-gt 是否大于
-lt 是否小于
-ge 是否大于等于
-le 是否小于等于

格式：

1test $a -eq $b  # a是否等于b
字符串比较参数 代表意义
test -z STRING 判断STRING是否为空，如果为空，则返回true
test -n STRING 判断STRING是否非空，如果非空，则返回true（-n可以省略）
test str1 == str2 判断str1是否等于str2
test str1 != str2 判断str1是否不等于str2
多重判断参数 代表意义
-a 是否同时成立
-o 是否至少一个成立
！ 取反

格式：

1test -r filename -a -x filename

判断符号[]

[]与test用法几乎一样，常用于if语句中，另外[[]]是[]的加强版，支持更多特性。

示例：

1[ 2 -lt 3 ]  # 为真，返回值为0
2echo $?  # 输出上个命令的返回值，输出0
3[ -e test.sh ] && echo "exist" || echo "Not exist"
4exist  # test.sh 文件存在
5[ -e test2.sh ] && echo "exist" || echo "Not exist"
6Not exist  # testh2.sh 文件不存在

注：

  • []每一项都要用空格隔开
  • 变量最好用双引号括起来
  • 常数最好用单或双引号括起来

示例：

1name="Uanu blog"
2[ $name == "Uanu blog" ]  # 错误，等价于 [ Uanu blog == "Uanu blog" ]，会报错：参数太多
3[ "$name" == "Uanu blog" ]  # 正确

文件重定向

每个进程都会默认打开3个文件描述符：

  • stdin标准输入，从命令行读取数据，文件描述符为0。
  • stdout标准输出，向命令行输出数据，文件描述符为1。
  • stderr标准错误输出，向命令行输出数据，文件描述符为2。

可以通过文件重定向将这三个文件重定向到其他文件中。

命令 说明
command > file 将stdout重定向到file中
command < file 将stdin重定向到file中
command » file 将stdout以追加方式重定向到file中
command n> file 将文件描述符n重定向到file中
command n» file 将文件描述符n以追加方式重定向到file中

示例：

1echo -e "Hello \c" > output.txt  # 将stdout重定向到output.txt中
2echo "World" >> output.txt  # 将字符串追加到output.txt中
3
4read str < output.txt  # 从output.txt中读取字符串
5
6echo $str  # 输出结果：Hello World

同时重定向stdin和stdout

1./test.sh < input.txt > output.txt
