hexo+github博客搭建

Posted by Uanu on Mon, Jan 1, 0001

一、本地环境准备

git安装

官方下载链接：https://git-scm.com/

1、直接下载安装即可，安装成功后右击鼠标会出现如图Git Bash Here的选项。

2、点击Git Bash Here打开Git的终端，输入git --version，如果显示git版本信息则说明安装成功。

Node.js安装

官方下载链接：https://nodejs.org/en/download/

1、直接下载安装即可。

2、安装后在刚才打开的Git终端中分别输入node -v和npm -v，如果显示Node和npm的版本信息则说明安装成功。

Hexo安装

1、新建存放博客的文件夹，打开文件夹并在文件夹内打开Git终端。

2、在Git终端中输入安装命令

1npm install hexo-cli -g

安装hexo。

3、在Git终端中输入安装命令

1npm install hexo-deployer-git --save

安装将hexo部署到github的deployer插件。

4、在Git终端中输入命令hexo -v，如果显示hexo的版本信息则说明hexo安装成功。

二、github准备

git配置ssh连接

上传博客或更新网站都需要将本地文件上传到github的仓库中，所以首先就需要将本地与github进行连接。这里使用的是ssh连接，ssh连接后续无需再次输入账号密码且上传更稳定。

1、生成ssh key 在终端中输入命令，双引号中为github绑定的邮箱。

1ssh-keygen -t rsa -C "xxxxxx@xxx.com"

然后直接连续回车三次（即按默认设置和保存），会显示生成的公钥，即图中蓝色框内。

公钥默认保存在C:\Users\DELL\.ssh文件夹中的id_rsa.pub文件中。

然后记事本打开文件复制内容（即公钥），或者终端输入命令直接复制。

1clip < ~/.ssh/id_rsa.pub

2、github添加公钥

github登录后，右上角打开setting。

然后找到SSH and GPG keys。

然后点击New SSH key绑定新的公钥，Title随便输，Key输入刚才复制的公钥。

最后点击Add SSH key完成。

3、在Git终端中输入命令。

1ssh -T git@github.com

接着输入yes，出现如图提示则连接成功。

4、配置Git个人信息 在Git终端输入命令配置用户名和邮箱，这是上传更新仓库时的身份凭证。

1git config --global user.name "用户名"
2git config --global user.email "邮箱"

三、博客搭建

本地搭建

1、初始化网站

在刚才创建存放博客的文件夹内打开Git终端，输入命令

1hexo init

hexo创建和初始化一个网站，文件夹中会出现各种文件。

2、生成静态网页

在Git终端中输入命令生成静态网页。

1hexo g

3、本地部署

在Git终端中输入命令在本地部署。

1hexo s

然后浏览器输入网址localhost:4000即可访问网站。ctrl+c可以关闭本地部署。

github部署

本地部署只能在hexo s运行时才能访问，而且别人无法通过互联网进行访问，所以我们需要将网站托管至github仓库。

1、创建仓库

点击New repository开始创建仓库。

仓库名称（repository name）必须是用户名.github.io，即和/前的用户名一样。其他设置默认即可。

然后点击Create repository完成创建。

2、修改配置文件

打开博客文件中的_config.yml，找到deploy选项，修改为

1deploy:
2	type: git
3	repository: SSH的链接
4	branch: main

其中，repository要先打开仓库，点击code，选择ssh，然后复制下面的链接。

branch要填仓库如图位置显示的分支。

然后保存配置文件。

3、发布到github

在Git终端输入命令将网站上传至仓库。

1hexo d

如果显示Deploy done且仓库出现文件，这说明上传成功。

如果出现报错ERROR Deployer not found: git

则先运行命令

1npm install hexo-deployer-git --save

部署成功后的网址为用户名.github.io，现在就可以任意时刻通过公网进行访问了（可能会有几分钟的延迟）。

四、博客使用

常用指令

1hexo new "postName" #新建文章
2hexo new page "pageName" #新建页面
3hexo g #生成静态页面至public目录
4hexo s #开启本地预览访问端口
5hexo d #部署到GitHub
6hexo help  # 查看帮助
7hexo v  #查看Hexo的版本

写博客及发布博客

写博客前首先要创建一篇博客，可以通过命令

1hexo new "postname"

然后source/_posts文件夹中就会出现一个名为postname.md的文件（当然也可以直接在该文件夹中新建md格式的文件），支持md语法（可以参照以往博客内容）。

博客开头要使用如下格式

1---
2title: first-blog
3date: 2022-08-12 12:00:00
4author: 
5tags: 
6categories: 
7---

两个---之间是该篇博客的属性设置，前两个属性title和date必须设置，其他属性（不止上述属性）可选。

上传博客是需要在Git终端中输入

1hexo clean # 清除缓存
2hexo g -d # 生成静态页面并上传至github

再次访问网站就可以看到更新的内容。

五、更换博客主题

有时基础的hexo博客功能可能较少，美观度也较差，这时可以通过更换主题来美化和优化网站。

1、下载主题

hexo官网主题：Themes | Hexo

在官网找到自己喜欢的主题并打开，然后找到该主题的github地址，一般在网页最底端。

然后进入该主题github仓库查看仓库地址。地址点击code即可查看，https和ssh均可。

然后在themes文件加打开Git终端，输入命令

1git clone 地址

themes文件夹中会出现该主题的文件夹。也可以直接点击Download ZIP下载该文件压缩包，解压后放在themes文件夹内。

2、修改配置文件

打开博客根目录的_config.yml文件，找到关键字theme，修改为

1themes: 主题文件名

3、更新网页

依次输入命令

1hexo clean
2hexo g -d

网页即可更换主题。

关于配置文件

博客根目录和主题文件夹内都存在_config.yml配置文件，前者是设置关于网站整体的设置（主题、插件等），后者是该主题的属性设置（如背景、颜色等）。

至此，一个简单的博客就算搭建完成了！

六、配置域名

如果我们想使用更加简单的域名访问网站，那么就需要配置域名。

1、在万网等平台购买域名。

2、打开域名控制台，点击解析。

然后选择下面任意一种方式添加记录：

  • 方法一：添加两个记录，两个记录类型都是 CNAME ，第一个主机记录为 @ ，第二个主机记录为 www，记录值都是填你自己的博客地址，即用户名.github.io。
  • 方法二：添加两个记录类型都为 A ，第一个主机记录为 @ ，第二个主机记录为 www，记录值都为博客的 IP 地址，IP 地址在cmd中输入ping 用户名.github.io获得。

3、接收域名 在网站source文件夹中添加名为CNAME的文件（无后缀），文件中填写域名（可不加www）。

然后就可以直接通过域名访问博客了。
