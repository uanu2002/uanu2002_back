KMP算法简介

KMP是三位大牛提出的一种字符串匹配算法。简单来说就是给定一个目标字符串，下称模式串（pattern string）；再给定一个待搜索的字符串，下称主串（main string），KMP就是在主串中定位模式串出现的位置。

算法分析

KMP算法的核心是不重复扫描主串，进而实现$O(n)$的时间复杂度。而要实现不重复扫描就必须准确得到每次模式串回溯的长度。下面将结合例子来推导回溯距离的计算。

我们似乎可以得到一个结论，当发生匹配失败（简称失配）时，我们可以保持主串i不动，将j移动至与失配前一位相同且距离最近的位置，然后j再加1；如果不存在则移至0。

第一种情况：和最开始一样分析，模式串和子串的4为匹配失败，由于模式串1位B和3位B间不存在B，所以可以直接将j移至1位B，然后加1。

第二种情况：此时如果按之前分析的方法，会出现如下图的情况，即主串2位的A与子串0位的C匹配失败，这是因为我们没有考虑与失配前两位相同且距离最近的位置。

综上考虑两种情况我们发现最开始得到的结论是错误的，事实上我们需要在确定j位置后，还需要验证是否直到模式串开始的部分都相同，因此我们需要修改结论为当发生匹配失败（简称失配）时，我们可以保持主串i不动，将j移动至模式串前n位与失配前n位相同且距离最近的位置，然后j再加1；如果不存在则移至0。

在得到结论后，我们下一步就要分析如何基于结论计算每个位置如果发生失配应该移动的距离，即求出next[i]数组。

计算next数组

计算next数组可以看做是模式串和模式串自身进行匹配，因此当发生失配时，我们可以直接使用前面计算好next进行移动。代码如下。

void Next(char*T){
    int i=1, j=0;
    while (i<strlen(T)) {
        if (j==0||T[i]==T[j]) {
            i++,j++;
            next[i]=j;
        }else
            j=next[j]; // i,j失配，用next[j]与i继续匹配
    }
}

next数组优化

考虑如下情况。

根据上述分析计算，出现失配后，j应该移动至1位的B，但由于失配的也是B，所以移动后必定失配，所以需要在求next数组时特别考虑p[j] == p[next[j]]的情况，即跳过相等情况的位置，优化后代码如下。

void Next(char*T){
    int i=1, j=0;
    while (i<strlen(T)) {
        if (j==0||T[i]==T[j]) {
            i++,j++;
            if(T[i] != T[j])
                next[i] = j;
            else
                next[i] = next[j]; // 跳过相同的位置
        }else
            j=next[j];
    }
}

注

上述的next数组均指失配位置的next数组，如果求的是失配前一位的next数组（例如下面代码），则无法使用常数优化。
