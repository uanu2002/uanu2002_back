「Git」Git Flow工作流

工具 git GitFlow
工具 工作流
Publish Date:   2022-09-18

工作流简介

工作流（WorkFlow）：工作流程及操作规则的概括描述。

Git Flow：是工作流的一种，它定义了项目的分支模型及各分支的作用，还定义了分支间的交互要求。

Git Flow

将代码托管平台上的库称为中央存储库origin。

主要分支

develop和master

中央存储库有两个无限生命周期的分支：

  • master
  • develop

这两个分支用于记录项目的历史版本，master保存正式发布的版本，develop则保存集成特性的分支（测试版本）。

feature

每一个特性都需要有自己的feature分支，当一个特性开发完成后，对应feature会被merge到develop分支中。feature将develop作为其父分支。不能直接与master合并。

release

发布分支release，当develop已经有足够多的新特性时，需要从develop分支开辟一个release分支。release分支创建后不允许新特性加入，只允许修复bug和发布任务。当release准备好后，将release分支与master分支合并，同时给master分支打上一个版本号。

hotfix

修补分支hotfix，用于快速修补master版本，是唯一从master开辟的分支。当hotfix修复完成后需要合并回master、develop和release分支。

工具包安装使用

为了更高效的实现上述操作，可以在git中下载git-flow，官网下载链接。

常用指令

  • git flow init：创建develop分支。
  • git flow feature start feature_branch：创建一个名为feature_branch的特征分支。
  • git flow feature finish feature_branch：结束一个名为feature_branch的特征分支（即与develop进行merge）。
  • git flow release start 0.1.0：创建一个名为0.1.0的release分支。
  • git flow release finish '0.1.0'：结束一个名为0.1.0的release分支（即与master进行merge）。
  • git flow hotfix start hotfix_branch：创建一个名为hotfix_branch的修补分支。
  • git flow hotfix finish hotfix_branch：结束一个名为hotfix_branch的修补分支（即与master、develop进行merge，然后删除该分支）。

补充

git flow指令与git指令并不冲突，实质上是将git指令进行集成。

总流程

总结下Git Flow的完整流程：

  1. 从 master 创建一个 develop 分支
  2. 从 develop创建一个 release 分支
  3. 从 develop 创建 feature 分支
  4. 当一个 feature 开发结束，将被 merge 到 develop 分支
  5. 当 release 分支完成，将被 merge 到 develop 和 master
  6. 如果在 master 中发现问题，要从 master 创建一个 hotfix 分支
  7. 一旦 hotfix 完成，将被 merge 回 develop 和 master
